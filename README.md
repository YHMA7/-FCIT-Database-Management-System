# Database Management System Project (FCIT)

This repository contains a full-scale database system project developed for the **CPIT-240: Database Management Systems** course at **King Abdulaziz University**. The project demonstrates the ability to design and implement a relational database from the ground up.

---

## 📂 Project Phases

### Phase 1: Requirement Analysis & Conceptual Design
* **System Requirements**: Detailed analysis of the system's goals and functional requirements.
* **ER/EER Modeling**: Design of the Enhanced Entity-Relationship diagram, identifying entities, attributes, and complex relationships.

### Phase 2: Logical Design & Schema Mapping
* **Relational Schema**: Mapping the EER diagram into a set of relational tables.
* **Normalization**: Ensuring data integrity and reducing redundancy through normalization techniques.
* **Functional Dependencies**: Defining dependencies to optimize table structures.

### Phase 3: Physical Implementation (SQL)
* **DDL (Data Definition Language)**: SQL scripts for creating tables, constraints, and relationships.
* **DML (Data Manipulation Language)**: SQL scripts for inserting sample data and performing core database operations.
* **Query Execution**: Testing the database with complex SQL queries to ensure performance and accuracy.

---

## 🛠️ Technologies & Tools
* **Database Engine**: Oracle SQL.
* **Modeling Tools**: Draw.io / Lucidchart (for ER/EER diagrams).
* **Documentation**: Detailed reports for each phase of the SDLC.
